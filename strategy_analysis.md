## 策略1: 简单RSI策略 (`simple_rsi`)
- **Python类**: [`SimpleRSIStrategy`](strategies.py:80)
- **描述**: 使用RSI指标判断超买超卖

### 可配置参数

#### 1. `rsi_period`
- **类型**: `int`
- **默认值**: `14`
- **描述**: RSI计算的周期长度。
- **含义与用途**: 定义了计算相对强弱指数（RSI）时回顾多少个时间单位（例如K线数量）的数据。
- **对策略表现的潜在影响**:
    - **较短周期 (例如 5-10)**: RSI对价格变化更敏感，产生更频繁的交易信号。这可能导致更高的交易频率，能更快捕捉短期趋势，但也可能引入更多市场噪音和假信号，从而增加交易成本和潜在亏损风险。回报波动可能更大。
    - **较长周期 (例如 20-30)**: RSI更平滑，对短期价格波动不那么敏感，产生的交易信号较少。这能有效过滤市场噪音，信号可能更可靠，但可能错过一些短期交易机会，且信号滞后性增加。交易频率降低，可能更适合捕捉中期或长期趋势。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'rsi_period': 14}

  # 配置为更敏感的RSI
  # strategy_params = {'rsi_period': 10}

  # 配置为更平滑的RSI
  # strategy_params = {'rsi_period': 20}
  ```

#### 2. `rsi_overbought`
- **类型**: `int`
- **默认值**: `70`
- **描述**: RSI超买阈值。
- **含义与用途**: 当RSI值超过此阈值时，认为市场可能处于超买状态，价格短期内可能回调下跌。策略通常在此条件下寻找卖出或平仓多头头寸的机会。
- **对策略表现的潜在影响**:
    - **较高阈值 (例如 80-90)**: 需要更强的市场超买信号才会触发，交易频率降低。这可能错过一些潜在的反转机会，但信号的可靠性可能更高，假信号减少。
    - **较低阈值 (例如 60-70)**: 更容易达到超买状态，交易频率增加。这能更早地识别潜在的超买情况，但也可能导致因市场短期波动而产生的假信号增多。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'rsi_overbought': 70}

  # 配置为更严格的超买条件
  # strategy_params = {'rsi_overbought': 80}
  ```

#### 3. `rsi_oversold`
- **类型**: `int`
- **默认值**: `30`
- **描述**: RSI超卖阈值。
- **含义与用途**: 当RSI值低于此阈值时，认为市场可能处于超卖状态，价格短期内可能反弹上涨。策略通常在此条件下寻找买入或平仓空头头寸的机会。
- **对策略表现的潜在影响**:
    - **较低阈值 (例如 10-20)**: 需要更强的市场超卖信号才会触发，交易频率降低。这可能错过一些潜在的反转机会，但信号的可靠性可能更高，假信号减少。
    - **较高阈值 (例如 30-40)**: 更容易达到超卖状态，交易频率增加。这能更早地识别潜在的超卖情况，但也可能导致因市场短期波动而产生的假信号增多。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'rsi_oversold': 30}

  # 配置为更严格的超卖条件
  # strategy_params = {'rsi_oversold': 20}
  ```

## 策略2: RSI+SMA策略 (`rsi_sma`)
- **Python类**: [`RSISMA_Strategy`](strategies.py:339)
- **描述**: 结合RSI指标和SMA均线判断趋势

### 可配置参数

#### 1. `rsi_period`
- (同策略1的 `rsi_period`)

#### 2. `rsi_overbought`
- (同策略1的 `rsi_overbought`)

#### 3. `rsi_oversold`
- (同策略1的 `rsi_oversold`)

#### 4. `sma_period`
- **类型**: `int`
- **默认值**: `30`
- **描述**: SMA均线周期。
- **含义与用途**: SMA（简单移动平均线）的计算周期。在此策略中，SMA用于判断当前的主要价格趋势。RSI的超买/超卖信号会结合当前价格与SMA的关系进行过滤。例如，一个RSI超卖信号，如果同时价格低于SMA（表明处于下降趋势），则可能被视为趋势延续中的回调结束信号，从而增加买入信号的置信度；反之，如果价格高于SMA，则可能被视为趋势不明朗或潜在的假信号。
- **对策略表现的潜在影响**:
    - **较短SMA周期 (例如 10-20)**: SMA对价格变化反应更快，更能代表短期趋势。这可能导致策略对短期趋势变化更敏感，交易频率可能增加。
    - **较长SMA周期 (例如 50-100)**: SMA反应较慢，更能代表中长期趋势，能有效过滤掉短期市场波动。这可能使策略信号更稳健，但可能滞后于趋势的早期变化。
    - SMA周期的选择会显著影响趋势判断的准确性和RSI信号的有效性。如果SMA周期太短，可能导致趋势判断不稳，增加错误信号；如果太长，则可能使策略反应迟钝，错过交易机会。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {
      'rsi_period': 14,
      'rsi_overbought': 70,
      'rsi_oversold': 30,
      'sma_period': 30
  }

  # 配置为使用更长期的SMA趋势过滤
  # strategy_params = {'sma_period': 50}
  ```

## 策略3: MACD策略 (`macd`)
- **Python类**: [`MACDStrategy`](strategies.py:241)
- **描述**: 使用MACD金叉死叉判断趋势转变

### 可配置参数

#### 1. `fast_period`
- **类型**: `int`
- **默认值**: `12`
- **描述**: 快线周期。
- **含义与用途**: MACD（异同移动平均线）计算中短期EMA（指数移动平均线）的周期。这条快线对近期价格变化反应较为灵敏。
- **对策略表现的潜在影响**:
    - **较短周期**: 使快线对价格变化更敏感，MACD线波动更频繁。
    - 与`slow_period`的差值决定了MACD指标的整体敏感度。如果`fast_period`和`slow_period`的差值较小，MACD线的波动会相对平缓，产生的交易信号可能较少。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'fast_period': 12}
  ```

#### 2. `slow_period`
- **类型**: `int`
- **默认值**: `26`
- **描述**: 慢线周期。
- **含义与用途**: MACD计算中长期EMA的周期。这条慢线相对平滑，更能代表中期趋势。MACD线本身是快线减去慢线得到的。
- **对策略表现的潜在影响**:
    - **较长周期**: 使慢线更平滑，更能反映长期趋势。
    - `fast_period`和`slow_period`的差值是关键。差值较大，MACD线波动幅度更大，可能产生更多交易信号，但也可能增加假信号。差值较小，MACD线更平滑，信号减少但可能更可靠。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'slow_period': 26}
  ```

#### 3. `signal_period`
- **类型**: `int`
- **默认值**: `9`
- **描述**: 信号线周期。
- **含义与用途**: MACD信号线（DEA或DEM）是MACD线的EMA。交易信号（如金叉、死叉）通常是基于MACD线与其信号线的交叉产生的。MACD柱状图（Histogram）是MACD线与信号线的差值。
- **对策略表现的潜在影响**:
    - **较短周期**: 使信号线更贴近MACD线，导致交叉信号更频繁，策略反应更快，但可能增加假信号。
    - **较长周期**: 使信号线更平滑，交叉信号减少，策略反应较慢，但信号可能更可靠，有助于过滤短期波动。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {
      'fast_period': 12,
      'slow_period': 26,
      'signal_period': 9
  }
  ```

## 策略4: 均线交叉策略 (`ma_cross`)
- **Python类**: [`MovingAverageCrossStrategy`](strategies.py:444)
- **描述**: 短期均线上穿/下穿长期均线产生信号

### 可配置参数

#### 1. `short_period`
- **类型**: `int`
- **默认值**: `9`
- **描述**: 短期均线。
- **含义与用途**: 短期移动平均线的计算周期，用于代表短期市场趋势。
- **对策略表现的潜在影响**:
    - **较短周期**: 对价格变化反应更快，更敏感。
    - 与`long_period`的配合决定了交叉信号的频率和滞后性。周期越短，对近期价格变动越敏感，可能导致更频繁的交易信号。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'short_period': 9}
  ```

#### 2. `long_period`
- **类型**: `int`
- **默认值**: `21`
- **描述**: 长期均线。
- **含义与用途**: 长期移动平均线的计算周期，用于代表长期市场趋势。`long_period`必须大于`short_period`。
- **对策略表现的潜在影响**:
    - **较长周期**: 反应较慢，均线更平滑，更能代表较稳定的长期趋势。
    - 短期均线和长期均线的**周期差**:
        - **差距较大**: 交叉信号频率较低，趋势确认可能更强，但信号滞后性也更大，可能错过趋势的早期阶段。
        - **差距较小**: 交叉信号频率较高，策略反应更快，但假信号的可能性也随之增加。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'long_period': 21}
  ```

#### 3. `ma_type`
- **类型**: `select` (选项: `SMA`, `EMA`)
- **默认值**: `EMA`
- **描述**: 均线类型。
- **含义与用途**: 选择使用SMA（简单移动平均线）还是EMA（指数移动平均线）来计算短期和长期均线。
- **对策略表现的潜在影响**:
    - **`SMA` (简单移动平均线)**: 对周期内所有价格数据点给予相同的权重。其走势相对平缓。
    - **`EMA` (指数移动平均线)**: 对近期的价格数据点给予更高的权重，因此EMA比SMA对价格变化的反应更灵敏。
    - 选择`EMA`可能导致策略更早地发出交易信号，从而更快地捕捉趋势，但也可能因为对短期波动的过度敏感而增加假信号的风险。选择`SMA`则信号相对滞后但可能更平稳。
- **配置示例**:
  ```python
  # 使用默认参数 (EMA)
  strategy_params = {
      'short_period': 9,
      'long_period': 21,
      'ma_type': 'EMA'
  }

  # 配置为使用SMA
  # strategy_params = {'ma_type': 'SMA'}
  ```

## 策略5: 布林带均值回归策略 (`bb_reversal`)
- **Python类**: [`BollingerBandsReversalStrategy`](strategies.py:536)
- **描述**: 价格触及布林带边界后反转回带内 (谨慎使用，均值回归型策略)

### 可配置参数

#### 1. `bb_period`
- **类型**: `int`
- **默认值**: `20`
- **描述**: 布林带周期。
- **含义与用途**: 布林带中轨（通常是N周期的SMA）和标准差的计算周期。
- **对策略表现的潜在影响**:
    - **较短周期**: 使布林带对价格变化更敏感，带宽能更快地根据近期波动率进行调整。
    - **较长周期**: 使布林带更平滑，带宽调整相对较慢，更能反映较长一段时间内的波动情况。
    - 周期选择影响布林带的整体形态和对价格波动的包裹程度。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'bb_period': 20}
  ```

#### 2. `bb_std_dev`
- **类型**: `float`
- **默认值**: `2.0`
- **描述**: 布林带标准差倍数。
- **含义与用途**: 定义布林带上轨和下轨与中轨之间的距离，即中轨加/减去N倍的标准差。
- **对策略表现的潜在影响**:
    - **较大标准差倍数 (例如 2.5-3.0)**: 布林带更宽。价格触及上下轨的次数减少，信号频率降低。触及边界可能意味着更极端的市场行情，反转信号的潜在强度可能更大。
    - **较小标准差倍数 (例如 1.5-2.0)**: 布林带更窄。价格触及上下轨更频繁，信号频率增加，但可能包含更多因正常市场波动而非真实超买/超卖造成的假信号。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'bb_std_dev': 2.0}

  # 配置为更宽的布林带
  # strategy_params = {'bb_std_dev': 2.5}
  ```

#### 3. `reversal_confirmation_candles`
- **类型**: `int`
- **默认值**: `1`
- **描述**: 价格回到带内确认K线数。
- **含义与用途**: 当价格从布林带外侧（例如低于下轨或高于上轨）回到布林带内侧后，需要多少根后续的K线来确认这个反转信号的有效性。
- **对策略表现的潜在影响**:
    - **`0`**: 价格一旦回到带内立即产生信号。这种方式反应最快，但可能因为价格只是短暂回调而产生假信号。
    - **`1`或更高**: 需要后续1根或多根K线收盘仍在带内（或继续向中轨运动）才确认信号。这增加了信号的确认度，有助于过滤掉一些假突破/假回归，但会引入信号滞后，可能错过最佳入场点。值越大，信号越滞后，交易频率越低，但理论上信号可靠性越高。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {
      'bb_period': 20,
      'bb_std_dev': 2.0,
      'reversal_confirmation_candles': 1
  }

  # 配置为不需要额外确认K线 (更激进)
  # strategy_params = {'reversal_confirmation_candles': 0}
  ```

## 策略6: 增强K线形态策略 (`enhanced_candlestick`)
- **Python类**: [`EnhancedCandlestickPatternStrategy`](strategies.py:805)
- **描述**: 识别吞没形态和Pin Bar (锤子/射击之星)

### 可配置参数

#### 1. `engulfing_min_body_ratio`
- **类型**: `float`
- **默认值**: `0.6`
- **描述**: 吞没K线实体最小占比。
- **含义与用途**: 定义了有效的吞没K线，其实体部分（开盘价与收盘价之差的绝对值）至少占其总长度（最高价-最低价）的这个比例。用于确保吞没K线本身是一个具有显著实体的K线，而不是影线过长、实体过小的K线。
- **对策略表现的潜在影响**:
    - **较高值 (例如 0.8)**: 要求吞没K线具有非常坚实的实体，这样的信号可能更强，但符合条件的形态会减少，导致交易频率降低。
    - **较低值 (例如 0.4)**: 允许实体相对较小的K线也被识别为吞没K线，符合条件的形态更多，但信号强度可能有所减弱。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'engulfing_min_body_ratio': 0.6}
  ```

#### 2. `engulfing_factor`
- **类型**: `float`
- **默认值**: `1.0`
- **描述**: 被吞没K线与吞没K线实体比例。
- **含义与用途**: 吞没K线的实体长度相对于被吞没K线（前一根K线）实体长度的最小倍数。用于定义吞没的强度。
- **对策略表现的潜在影响**:
    - **大于1.0的值 (例如 1.2)**: 要求吞没K线的实体显著大于前一根K线的实体，这样的吞没信号被认为更强，但符合条件的形态会减少。
    - **等于1.0或略小于1.0的值**: 条件相对宽松，更多形态会被识别，但信号强度可能不一。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'engulfing_factor': 1.0}
  ```

#### 3. `pinbar_min_wick_body_ratio`
- **类型**: `float`
- **默认值**: `2.0`
- **描述**: PinBar影线/实体最小比例。
- **含义与用途**: Pin Bar（锤子线或射击之星）的主要影线（例如看涨Pin Bar的下影线，或看跌Pin Bar的上影线）长度与其实体长度的最小比例。用于定义Pin Bar影线的显著性，影线越长，通常反转信号的指示意义越强。
- **对策略表现的潜在影响**:
    - **较高值 (例如 3.0)**: 要求Pin Bar具有非常长的影线相对于其实体，这样的信号被认为更强，但符合条件的形态会较少。

## 策略7: 价格突破策略 (`price_breakout`)
- **Python类**: [`PriceBreakoutStrategy`](strategies.py:680)
- **描述**: 价格突破N周期高点/低点产生信号

### 可配置参数

#### 1. `lookback_period`
- **类型**: `int`
- **默认值**: `20`
- **描述**: 回顾周期N。
- **含义与用途**: 用于确定前期高点/低点的回顾周期。例如，如果设置为20，则策略会关注过去20根K线的最高价和最低价作为潜在的突破水平。
- **对策略表现的潜在影响**:
    - **较短周期 (例如 10)**: 对近期价格形成的支撑/阻力位敏感，突破信号可能更频繁，但这些水平可能不够稳固，导致假突破增多。
    - **较长周期 (例如 50)**: 形成的支撑/阻力位通常更强，被市场验证的时间更长。突破这些水平的信号较少，但一旦发生，可能代表更重要的趋势变化或延续。交易频率会降低，但信号可靠性可能更高。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'lookback_period': 20}

  # 配置为关注更长期的突破水平
  # strategy_params = {'lookback_period': 50}
  ```

#### 2. `volume_confirmation_factor`
- **类型**: `float`
- **默认值**: `1.2`
- **描述**: 成交量确认因子。
- **含义与用途**: 突破发生时，当前K线的成交量必须至少是回顾周期（`lookback_period`）内平均成交量的这个倍数，才认为突破得到了成交量的确认。
- **对策略表现的潜在影响**:
    - **较高因子 (例如 1.5-2.0)**: 要求突破时有显著放大的成交量。这样的信号通常被认为更可靠，因为成交量支持了价格的突破行为。符合条件的突破会减少，交易频率降低。
    - **较低因子 (例如 1.0或更低)**: 对成交量的要求不高，甚至不要求放量。信号更多，但可能包含一些没有成交量支持的“无力”突破，这些突破失败的风险较高。
    - 如果数据中不包含成交量信息，或成交量数据质量不高，此参数的有效性会降低。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {
      'lookback_period': 20,
      'volume_confirmation_factor': 1.2
  }

  # 配置为需要更强的成交量确认
  # strategy_params = {'volume_confirmation_factor': 1.5}
  ```

## 策略8: 随机指标策略 (`stochastic_oscillator`)
- **Python类**: [`StochasticOscillatorStrategy`](strategies.py:930)
- **描述**: 使用随机指标的超买超卖及金叉死叉

### 可配置参数

#### 1. `k_period`
- **类型**: `int`
- **默认值**: `14`
- **描述**: %K周期。
- **含义与用途**: 随机指标中%K线的计算周期。%K线衡量当前收盘价在过去N（`k_period`）周期内的价格范围（最高价与最低价之间）中所处的位置。
- **对策略表现的潜在影响**:
    - **较短周期**: 使%K线对价格变化更敏感，波动更剧烈，产生更多交易信号。
    - **较长周期**: 使%K线更平滑，波动减缓，信号频率降低但可能更稳定。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'k_period': 14}
  ```

#### 2. `d_period`
- **类型**: `int`
- **默认值**: `3`
- **描述**: %D周期。
- **含义与用途**: %D线是%K线的M（`d_period`）周期移动平均。它作为%K线的平滑线，用于产生交叉信号（%K上穿或下穿%D）。
- **对策略表现的潜在影响**:
    - **较短周期**: 使%D线更贴近%K线，导致交叉信号更频繁，策略反应更快。
    - **较长周期**: 使%D线更平滑，与%K线的交叉信号减少，策略反应较慢，但有助于过滤掉一些噪音。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'d_period': 3}
  ```

#### 3. `slowing`
- **类型**: `int`
- **默认值**: `3`
- **描述**: %K平滑周期。
- **含义与用途**: 在慢速随机指标（Slow Stochastic）中，原始的快速%K线（Fast %K）会先经过一个S（`slowing`）周期的移动平均，得到慢速%K线（Slow %K）。然后慢速%K线再经过`d_period`的移动平均得到慢速%D线。此参数用于平滑快速%K线。
- **对策略表现的潜在影响**:
    - **`1`**: 表示不进行额外的平滑，此时慢速%K等于快速%K（如果策略内部是这样处理的话，或者说快速%K直接用于计算%D）。
    - **较大的值 (例如 3-5)**: 会使慢速%K线（以及后续的%D线）更加平滑，减少指标的毛刺和噪音，从而可能产生更可靠的信号，但同时也会增加信号的滞后性。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'slowing': 3}
  ```

#### 4. `overbought`
- **类型**: `int`
- **默认值**: `80`
- **描述**: 超买阈值。
- **含义与用途**: 当随机指标（通常指%K或%D线）超过此值时，认为市场可能处于超买状态。
- **对策略表现的潜在影响**:
    - **较高阈值 (例如 85-90)**: 需要更强的市场超买信号才会触发，交易频率降低，但信号可能更可靠。
    - **较低阈值 (例如 70-75)**: 更容易达到超买状态，交易频率增加，但假信号风险也随之增加。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'overbought': 80}
  ```

#### 5. `oversold`
- **类型**: `int`
- **默认值**: `20`
- **描述**: 超卖阈值。
- **含义与用途**: 当随机指标（通常指%K或%D线）低于此值时，认为市场可能处于超卖状态。
- **对策略表现的潜在影响**:
    - **较低阈值 (例如 10-15)**: 需要更强的市场超卖信号才会触发，交易频率降低，但信号可能更可靠。
    - **较高阈值 (例如 25-30)**: 更容易达到超卖状态，交易频率增加，但假信号风险也随之增加。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {
      'k_period': 14,
      'd_period': 3,
      'slowing': 3,
      'overbought': 80,
      'oversold': 20
  }
  ```

## 策略10: 布林带挤压突破(剥头皮) (`bb_squeeze_breakout`)
- **Python类**: [`BBSqueezeBreakoutStrategy`](strategies.py:1154)
- **描述**: 在布林带收窄（挤压）后，等待价格向某个方向突破，适用于剥头皮

### 可配置参数

#### 1. `bb_period`
- (同策略5的 `bb_period`)

#### 2. `bb_std_dev`
- (同策略5的 `bb_std_dev`)

#### 3. `squeeze_threshold_pct`
- **类型**: `float`
- **默认值**: `0.5`
- **描述**: 挤压阈值(带宽/中轨%)。
- **含义与用途**: 布林带宽度（定义为 (上轨 - 下轨) / 中轨 * 100）必须低于此百分比阈值，才认为布林带处于“挤压”（Squeeze）状态。挤压状态通常表示市场波动率降低，可能预示着未来将有一次较大幅度的价格波动（突破）。
- **对策略表现的潜在影响**:
    - **较低阈值 (例如 0.2%-0.4%)**: 要求布林带非常窄，挤压状态更明显，这样的信号可能更强，预示的突破力度可能更大，但发生的频率会较低。
    - **较高阈值 (例如 0.8%-1.5%)**: 布林带相对较宽时也可能被视为挤压，信号频率较高，但可能并非真正的强力挤压，突破的可靠性或幅度可能较低。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'squeeze_threshold_pct': 0.5}
  ```

#### 4. `breakout_confirmation_candles`
- **类型**: `int`
- **默认值**: `1`
- **描述**: 突破确认K线数。
- **含义与用途**: 在识别到布林带挤压状态之后，实际的突破K线是在挤压状态发生的K线之后的第N（`breakout_confirmation_candles`）根K线。代码实现中，通常是检查当前K线之前的`breakout_confirmation_candles`根K线是否处于挤压状态，然后看当前K线是否形成突破。
- **对策略表现的潜在影响**:
    - **`1`**: 检查紧邻的前一根K线是否处于挤压状态，然后判断当前K线是否突破了该挤压状态下的布林带边界。
    - **增加此值**: 意味着挤压状态需要持续更长的时间（或突破发生在挤压形成后的稍晚时间点），才考虑其突破信号。这可能有助于过滤掉一些短暂的挤压或不稳定的突破，但更强调挤压的持续性和突破的延后确认。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {'breakout_confirmation_candles': 1}
  ```

#### 5. `min_breakout_body_pct`
- **类型**: `float`
- **默认值**: `0.3`
- **描述**: 突破K线最小实体占比。
- **含义与用途**: 发生突破的K线，其实体部分（开盘价与收盘价之差的绝对值）必须至少占其总长度（最高价-最低价）的这个百分比。用于确保突破是由一根具有明确方向和力度的K线完成的，而不是犹豫不决的十字星或小实体K线。
- **对策略表现的潜在影响**:
    - **较高值 (例如 0.5-0.7)**: 要求突破K线具有非常坚实的实体，这样的突破信号被认为更强，但符合条件的突破会减少，交易频率降低。
    - **较低值 (例如 0.1-0.3)**: 允许实体相对较小的K线也被视为有效突破，交易机会更多，但信号强度和可靠性可能有所下降。
- **配置示例**:
  ```python
  # 使用默认参数
  strategy_params = {
      'bb_period': 20,
      'bb_std_dev': 2.0,
      'squeeze_threshold_pct': 0.5,
      'breakout_confirmation_candles': 1,
      'min_breakout_body_pct': 0.3
  }
